My Experience Hosting My Own Server

A few days after New Year's, I decided to get a VPS. I started by hosting a Discord bot, and it went well. However, after the webhook I used to keep track of activity on my website was deleted, I decided to create a backend for my website to prevent that from happening again. This resulted in an issue since the VPS didn’t have HTTPS but my website did. So, I decided to move all of the hosting over to my VPS and ditch HTTPS. Later, I got HTTPS using Certbot, so in hindsight, the earlier effort a waste of time. Still, I’m glad I did it. Hosting my own website has been a fun experience, and I’ve learned quite a lot.

Advantages

Hosting my website on my own server gives me much more control. I can customize the error pages and use subdomains. GitHub Pages is great, but I have so much more freedom to fine-tune everything. Another advantage is that the server doesn’t have to be used solely for a website—it can also function as a VPN, NAS, proxy, etc.

Disadvantages

It costs money, although the amount depends on how you’re hosting. This VPS costs only about $5 a month and has more resources than I could ever need. I am also responsible for the security of the website and server, so it’s my job to ensure my server isn’t attacked. Additionally, it requires more work and experience since everything on the server is purely command-line-based.

Summary

Using my own server for hosting a website, despite the disadvantages, has been well worth it. However, if all you want is something simple, GitHub Pages would work just fine.
